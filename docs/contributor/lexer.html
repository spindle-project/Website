From Text to Tokens: The Lexer

The Lexer is the first stage (ignoring the semi-parser) of converting Spindle code into a result. In summary, it takes your text and condenses it into tokens.
The lexer goes through your text â€” character by character (ignoring tabs and spaces, but taking note of newlines). In a sense, it translates your text file into a more readable term for later steps. There is some minor error checking at this stage, as it ensures you use the right syntax for simple things. The process described above occurs in the make_tokens() function. This: 

a <-- 10 + 5

Is transformed into something like this: [IDENTIFIER:a, EE, INT:10, PLUS, INT:5] The lexer also defines an advance() function that moves the current token to the next one.
